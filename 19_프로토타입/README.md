- 자바스크립트는 명령형, 함수형, (              )을 지원하는 멀티 패러다임 프로그래밍 언어다. 
객체지향 프로그래밍은 여러개의 독립적인 (    )의 집합으로 프로그램을 표현하려는 프로그래밍 패러다임이다. 객체는 (   )와 (   )를 하나로 묶은 복합적인 자료구조라 할 수 있다.
    
    프로토타입기반 객체지향 프로그래밍, 객체, 프로퍼티, 메서드
    
- ( A ) 은 객체 지향 프로그래밍의 핵심 개념으로 객체의 프로퍼티, 메서드를 다른 객체가 ( A )받아 그대로 사용할 수 있는 것을 말한다. ( B )을 기반으로 ( A )을 구현하여 불필요한 중복을 제거한다.
    
    A 상속, B 프로토타입
    
- 객체 리터럴로 객체를 생성하는 것과, 생성자 함수로 인스턴스를 생성하는 것 모두 생성자함수와 연결되어 있다. 프로토타입은 ( A ) 가 생성되는 시점에 더불어 생성된다. 
(1)사용자정의 생성자 함수( +new연산자) 와 (2)빌트인 생성자함수의 프로토타입은 각각 언제 생성되는가?
    
    A : 생성자함수 
    
    (1) new연산자를 통해 생성자 함수를 만들때(277p)
    런타임 이전 생성자 함수가 평가되어 객체가 생성되는 시점에 프로토타입도 같이 생성됨, 이때 프로토타입은 object.prototype이다
    
    (2) 빌트인 생성자함수는 전역객체가 생성되는 시점에 프로토타입이 생성된다.  
    
- *객체는 추상연산에 의해 생성된다는 공통점이 있다.*
    
    추상 연산은 자바스크립트 스펙 문서에서 객체 생성, 프로퍼티 접근, 함수 호출 등과 같은 여러 동작을 정의하는데 사용됩니다.
    
    1. 새로운 객체 생성: 빈 객체가 생성됩니다.
    2. 프로토타입 연결: 생성된 객체의 [[Prototype]] 내부 슬롯에 생성자 함수의 prototype 프로퍼티를 가리키는 링크가 설정됩니다.
    3. 생성자 함수 호출: 생성자 함수가 호출되며, this 키워드는 새로 생성된 객체를 가리킵니다.
    4. 객체 반환: 생성된 객체가 반환됩니다.
    
    이러한 추상 연산을 통해 객체 생성이 이루어지며, 생성된 객체는 생성자 함수의 prototype 프로퍼티와 연결된 프로토타입 객체를 상속받습니다. 이렇게 생성된 객체는 프로퍼티와 메서드를 가지는 일반 객체가 되며, 프로토타입 체인을 통해 상위 객체의 프로퍼티와 메서드에도 접근할 수 있습니다.
    
- 객체가 생성되는 방식에 따라 프로토타입도 결정된다. (1) 객체리터럴 (2)object생성자함수 (3) 생성자함수의 프로토타입이 각각 어떻게 결정되는가?
    
    (1,2) 객체 리터럴과 Object생성자 함수로 생성한 객체는 Object.prototype을 프로토타입으로 갖게 되고, Object.prototype을 상속받아 사용할 수 있다. 
    
    (3) 사용자 정의 생성자함수로 생성한 객체 me는 Person.prototype의 프로퍼티인 constructor만 상속받아 사용할 수 있다. 하지만 프로토타입 체인으로 인해서 Object.prototype의 프로퍼티와 메서드도 상속받아 사용할 수 있다.

- 프로토타입 체인은 무엇인가
    
    객체의 프로퍼티를 사용하려고 할때, 해당 객체에 프로퍼티가 없으면 프로토타입으로 연결되어 있는 자신의 부모 객체를 참조하여 해당 프로퍼티를 검색한다. 이를 프로토타입 체인이라고 한다. 프로토타입 체인은 상속과 프로퍼티 검색을 위한 메커니즘이다
    

프로토타입은 상위 객체의 프로퍼티를 사용할 수 있게 하는 자바스크립트의 기술 중 하나. ㅋㅋ
