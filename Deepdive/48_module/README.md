# 모듈
자바스크립트에서 모듈은 코드를 구성하고 관리하는 데 사용되는 개념이다.<br> 모듈은 애플리케이션을 구성하는 개별적인 요소로서 재사용 가능한 코드 조각을 뜻한다. 모듈은 기능을 기준으로 파일 단위로 분리한다. 모듈은 자신만의 파일 스코프를 가질 수 있어야 한다.


### 모듈의 특징
- 코드의 분리 : 관련된 코드를 그룹화하여 독립적인 단위로 분리한다
- 캡슐화 : 내부 구현의 세부 사항을 숨기고 공개된 API를 통해서만 상호작용할 수 있도록 제한한다. 모듈에 포함돼 있는 변수, 함수, 객체 등은 캡슐화되어 다른 모듈에서 접근할 수 없다.
- 네임 스페이스 충돌 방지 : 모듈을 사용하여 변수, 함수 및 클래스 이름이 충돌하지 않도록 보호할 수 있다. 각 모듈은 자체 스코프를 가지기 때문에 그 안에서 정의된 식별자에 접근할 수 있다.

### export와 import 
- 모듈은 개별적 존재로서 애플리케이션과 분리돼 존재한다. 하지만 애플리케이션에서 사용할 수 있어야 모듈의 존재 의미가 있기 때문에 export라는 기능을 통해 모듈을 공개할 수 있다.

- ES6(ES2015)부터 자바스크립트는 공식적으로 모듈 시스템을 지원하며, **`import`**와 **`export`** 키워드를 사용하여 모듈을 정의하고 내보내기(export) 및 가져오기(import)할 수 있다.
